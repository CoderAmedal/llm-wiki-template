# 🧠 AI Second Brain

> Stop using AI to write code. Start using AI to **think better**.

基于 Andrej Karpathy 的实践，这个项目探索一种新的 AI 使用方式：
**不是让 AI 替你写代码，而是让 AI 帮你构建一个“第二大脑”（Second Brain）**。

---

## ✨ 核心理念

传统使用方式：

```
问问题 → 得到答案 → 复制粘贴 → 忘记
```

Karpathy 的方式：

```
输入原始信息 → AI 自动整理 → 构建知识库 → 持续进化
```

👉 从「工具」变成「长期知识系统」

---

## 🧠 什么是 Second Brain？

Second Brain = 一个 AI 驱动的个人知识系统：

* 📚 存储所有输入（文章 / 代码 / 想法）
* 🔍 可搜索（语义搜索，而不是关键词）
* 🔗 自动建立知识连接
* 🤖 可对整个知识库进行 AI 问答

本质是：
**把“记忆”和“理解”外包给 AI** ([Stack Overflow 博客][1])

---

## 🚀 项目目标

* 构建一个 **自维护知识库（self-maintaining knowledge base）**
* 用 AI 自动：

  * 分类
  * 总结
  * 建立关联
* 支持长期积累，而不是一次性对话

## 🏗️ 系统架构

参考 Karpathy 的实践：

```
                ┌──────────────┐
                │ 原始输入数据 │
                │ (文章/代码)  │
                └──────┬───────┘
                       ↓
              ┌────────────────┐
              │   LLM 处理层   │
              │ (总结/分类/重写)│
              └──────┬─────────┘
                     ↓
        ┌──────────────────────────┐
        │   知识库（Markdown/Wiki）│
        │   Obsidian / Git / DB    │
        └──────┬───────────────────┘
               ↓
        ┌──────────────────────────┐
        │ 查询 & 推理（AI Chat）    │
        └──────────────────────────┘
```

关键点：

* AI 不只是回答问题
* **AI 在维护整个知识系统** ([Tech Upkeep][2])

## 🧩 核心工作流

### 1️⃣ Ingest（摄入）

将新来源放入 `raw/` 目录，告诉 LLM 处理它：

```
raw/article.md → LLM 阅读 → 更新 wiki 多个页面（摘要、实体页、概念页、索引、日志）
```

单次摄入可能涉及 10-15 个 wiki 页面的更新。

### 2️⃣ Query（查询）

直接向 LLM 提问，例：

> "我之前关于数据库设计的所有知识有什么规律？"

LLM 先读 `index.md` 定位相关页面，再综合回答。好的答案可以沉淀为新的 wiki 页面。

### 3️⃣ Lint（检查）

定期让 LLM 检查 wiki 健康度：

- 页面间矛盾
- 过时信息（被新来源替代）
- 孤立页面（无 inbound 链接）
- 缺失交叉引用

---

## 📁 目录结构

```
.
├── raw/                    # 原始来源（不可变）
├── wiki/                   # LLM 生成的 wiki 页面
├── index.md                # wiki 内容索引
├── log.md                  # 操作日志（append-only）
└── AGENTS.md               # LLM 的工作schema
```

---

## 🔧 实际使用

```
# 1. 添加来源
把 article.md 放入 raw/

# 2. 让 LLM 处理
"请处理 raw/article.md，提取关键概念并更新 wiki"

# 3. LLM 会自动：
   - 写入 wiki/ 摘要页
   - 更新相关实体/概念页
   - 更新 index.md
   - 在 log.md 记录

# 4. 查询
"关于数据库设计我都知道些什么？"

# 5. 检查
"检查一下 wiki 是否有矛盾或过时信息"
```
